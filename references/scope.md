# §3 SCOPE — 어디까지 조사하는가

리서치 경계 정의·자원 제약 관리·종료 판정. 자원은 유한하다.

**트리거 연동:**
- **트리아지** (자원제약·긴급×중요·즉시/예약/위임/제거) — 깊이 판정·종료 판정
- **스켈레톤** + **제출청소** — 종료 게이트 정제
- **오컴** — 신뢰도 3요소 이진 판정

> [!info] LIGHT 모드
> - 3-1: 1줄 "안 보는 것" 메모
> - 3-2: 1줄 "못 본 것" 메모
> - 3-3: 전 축 scan 고정
> - 3-4: 면제 — 축 순회 완료 시 종료
> - 3-5: 축당 "높음/낮음" 1단어. LIGHT cap 50 (INV 4)

---

## 3-1. 스코프 경계 (사전)

리서치 착수 전 "무엇을 조사하지 않는가" 명시 — 경계 없으면 무한 팽창.

**설정 기준:**
- WHY 1-1 연동: 의사결정 무관 영역 = 스코프 밖
- "조사하지 않는 이유" 1줄 기록
- 도중 스코프 밖이 중요해지면 → 형 보고 + 확장 승인

**§5 EXPAND 사전승인 (v2.4):**
- T1·T2·T3 토큰은 §1-3 프레임 설계 시점에 **사전 예산 선언** 시 사용 시 추가 승인 면제
- 예산 초과·무선언 사용 = 형 승인 필수
- §4 연쇄 발급 토큰도 사전승인 범주 (별도 예산, INV 8)

**예시:**
```
스코프 안: 국내 시장, 경쟁 3사, 규제
스코프 밖: 해외 시장 (의사결정이 국내 진입이므로) — 해외 사례는 참고용 허용
```

---

## 3-2. 관찰 한계 기록 (사후)

종료 시 "무엇을 볼 수 없었는가" 기록 — 신뢰도 판단 핵심.

**기록 항목:**
- 접근 불가 소스 (페이월·비공개·내부)
- 시간 부족 미탐색
- 도구 한계 (검색 불가·언어)
- 각 한계의 결론 영향 추정 (치명적/경미)

---

## 3-3. 자원 제약 + 깊이 판정 (트리아지 발동)

**트리아지** 발동 — 시간·접근·비용 제약 사전 선언 + 축별 조사 깊이 결정.

**축별 깊이 3단계:**

| 깊이 | 정의 | 기준 |
|------|------|-----------|
| 전수(full) | 가용 소스 전수 + 포화 | 의사결정 영향↑ + 불확실성↑ |
| 표본(sample) | 주요 3-5개 + 교차 | 영향 중간 or 일부 알려진 |
| 스캔(scan) | 핵심 팩트만 빠르게 | 영향↓ or 보조 축 |

**판정 매트릭스 (VOI):**

| | 불확실성 높음 | 불확실성 낮음 |
|---|---|---|
| **영향 높음** | 전수 | 표본 |
| **영향 낮음** | 표본 | 스캔 |

**EVSI 간이 테스트 (추가 조사 지속 판정):**
"지금 멈추면 의사결정 바뀔 확률?" → 높음(>30%) = 계속 / 낮음(≤30%) = 종료. §3-4 조기종료와 동일 기준.

**맥가이버 활용:** 접근불가 소스 있을 때 비표준 경로(공식 막히면 채용공고·특허·납품사 역추적).

형이 별도 지정하면 우선.

---

## 3-4. 종료 게이트 (스켈레톤+제출청소)

**정상종료 3조건:**
1. 전 축 소스 포화 (깊이별)
2. 핵심 가설 전부 검증/반증 (경쟁 가설 포함)
3. L2 교차 치명 빈자리 0건

**포화 판정표:**

| 깊이 | 포화 요건 |
|------|---------------|
| 전수 | 장르 3종+ 커버 + 새 소스 2회 연속 결론 불변 (장르 다를 것) + 반대증거 완료 |
| 표본 | 장르 2종+ + 새 소스 1회 결론 불변 |
| 스캔 | 핵심 팩트 확인 (포화 면제) |

**조기종료:** 자원 소진 시 아래 충족하면 "충분히 좋은" 판정:
- WHY 1-1 답이 방향성 수준 존재
- 미탐색 영역 §3-2 명시
- 형에게 "여기까지 결론 + 추가 시 바뀔 부분" 보고

미충족 → 추가 조사 영역 명시 + 형 판단 요청.

**종료 필수 제출물:** §3-2 관찰 한계 + §3-5 축별 신뢰도. **제출청소** + **스켈레톤**으로 최종 정제.

---

## 3-5. 축별 신뢰도 등급

각 축 결론에 신뢰도 태깅 — 모든 축이 동일 무게가 아님.

> [!warning] 모드 분기
> - **LIGHT:** 3요소 분해 전체 면제. 축당 "높음/낮음" 1단어. 확신도 **50 cap** (INV 4).
> - **DEEP:** 3요소 분해 + 매핑 테이블 + 개수 세기 검증 풀 수행.

| 요소 | 높음 | 낮음 | 입력 (v2 직결) |
|------|------|------|---------------------|
| 소스 다양성 | 3종+ | 1종 | **§2-1 수렴4축 직결** — 3축+ = 높음, 2축 이하 = 낮음. 재계산 ✗ |
| 포화 달성 | 달성 | 미달 | **§2-1 포화 직결** |
| 반대증거 | 탐색 후 반증 실패(강화) | 미탐색 or 반증 성공 | **§2-2 허브 직결** — 재탐색 ✗ |

**v2 직결 원칙:** 3요소 모두 §2 산출값 재참조만. 재계산·재탐색 = 중복 회계 FAIL.

**FIG 통과 전제:** §2-3 FIG 3축 통과 팩트만 입력. 미통과 팩트는 계산 배제.

**확신도 태깅 (오컴: 이진 판정):**
- 90/70/50/30 중 하나만. 중간값(65·75·80·85) 금지. 100점 척도 금지.
- 3요소 각각 이진(높음/낮음). "중간·부분" 금지. 애매하면 낮음.
- 높음 개수 → 확신도:

| 높음 개수 | 확신도 |
|-----------|--------|
| 3 | 90 |
| 2 | 70 |
| 1 | 50 |
| 0 | 30 |

- 형식: `확신도: [90/70/50/30] — 소스다양성 [높음/낮음], 포화 [높음/낮음], 반대증거 [높음/낮음]`

**확신도 태깅 위치:** §3-5에서만. §1·§2 본문에서 "(확신도: 80)" 자유 점수 금지. "확신도" 단어 자체를 §3-5 외부에서 수치와 함께 사용 금지. 본문은 서술형("포화 달성·소스 3종 일치").

**개수 세기 검증:** 판정 후 반드시 "소스다양성=[높음/낮음], 포화=[높음/낮음], 반대증거=[높음/낮음] → 높음 N개" 나열 후 매핑.
