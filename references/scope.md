# §3 SCOPE — 어디까지 조사하는가

리서치의 경계를 정의하고, 자원 제약을 관리하고, 종료를 판정한다. 자원이 유한하다는 것은 자명한데, 이에 대한 구조적 대응이 필요하다.

> [!info] LIGHT 모드
> LIGHT에서 §3 SCOPE는 아래와 같이 경량화된다. 상세 절차는 DEEP 전용.
> - **3-1 스코프 경계:** 1줄 "안 보는 것" 메모
> - **3-2 관찰 한계:** 1줄 "못 본 것" 메모
> - **3-3 깊이 판정:** 전 축 scan 고정 (차등 배분 안 함)
> - **3-4 종료 게이트:** 면제 — 축 순회 완료 시 종료
> - **3-5 축별 신뢰도:** 축당 "높음/낮음" 1단어만 (3요소 분해 면제). LIGHT 확신도 상한 = 50

---

## 3-1. 스코프 경계 (사전)

리서치 착수 전에 "무엇을 조사하지 않는가"를 명시한다 — 경계가 없으면 리서치가 무한 팽창한다.

**설정 기준:**
- WHY 1-1 연동: 의사결정에 무관한 영역 = 스코프 밖
- "조사하지 않는 이유" 1줄 기록 (나중에 "왜 안 봤지?"에 대한 답)
- 도중 스코프 밖이 중요해지면 → 형에게 보고 + 확장 승인 요청

**예시:**
```
스코프 안: 국내 시장 규모, 경쟁자 3사, 규제 현황
스코프 밖: 해외 시장 (의사결정이 국내 진입 여부이므로) — 단, 해외 사례는 참고용 허용
```

---

## 3-2. 관찰 한계 기록 (사후)

리서치 종료 시 "무엇을 볼 수 없었는가"를 기록한다 — 결론의 신뢰도를 판단하는 핵심 정보.

**기록 항목:**
- 접근 불가 소스 (페이월, 비공개 데이터, 내부 문서 등)
- 시간 부족으로 미탐색 영역
- 도구 한계 (검색 불가, 언어 장벽 등)
- 각 한계가 결론에 미치는 영향 추정 (치명적/경미)

---

## 3-3. 자원 제약 + 깊이 판정

시간·접근·비용 제약을 사전 선언하고, 축별로 조사 깊이를 결정한다 — 모든 축을 동일 깊이로 파는 것은 비효율적.

**축별 깊이 3단계:**

| 깊이 | 정의 | 적용 기준 |
|------|------|-----------|
| 전수(full) | 가용 소스 전수 탐색 + 포화 판정까지 | 의사결정 영향↑ + 불확실성↑ |
| 표본(sample) | 주요 소스 3-5개 + 교차 확인 | 영향 중간 or 일부 알려진 영역 |
| 스캔(scan) | 핵심 수치·팩트만 빠르게 확인 | 영향↓ or 보조 축 |

**판정 매트릭스 (VOI 원칙 반영):**

| | 불확실성 높음 | 불확실성 낮음 |
|---|---|---|
| **의사결정 영향 높음** | 전수(full) — 추가 조사의 기대가치 최대 | 표본(sample) — 확인용 |
| **의사결정 영향 낮음** | 표본(sample) — 알면 좋지만 필수 아님 | 스캔(scan) — 최소 투자 |

**추가 조사 계속 여부 판정 (EVSI 간이 테스트):**
"지금 조사를 멈추면 의사결정이 바뀔 확률은?" → 높음(>30%) = 계속 / 낮음(≤30%) = 현 수준 종료. 이 판정은 §3-4 종료 게이트의 조기종료 판단에도 사용한다.

형이 별도 지정하면 우선.

---

## 3-4. 종료 게이트

**정상종료 3조건:**
1. 전 축 소스 포화 (깊이에 맞는 수준) — 아래 포화 판정표 참조
2. 핵심 가설 전부 검증 또는 반증 (경쟁 가설 포함)
3. L2 교차에서 치명적 빈자리 0건

**포화 판정표 (operational 기준):**

| 깊이 | 포화 선언 요건 |
|------|---------------|
| 전수(full) | 소스유형 3종+ 커버 + 새 소스 2회 연속 결론 불변 (유형 다를 것) + 반대증거 탐색 완료 |
| 표본(sample) | 소스유형 2종+ 커버 + 새 소스 1회 결론 불변 |
| 스캔(scan) | 핵심 수치/팩트 확인 완료 (포화 판정 면제) |

**조기종료:** 자원(시간) 소진 시 아래를 충족하면 "충분히 좋은" 판정:
- 의사결정 연결점(WHY 1-1)에 대한 답이 방향성 수준으로 존재
- 미탐색 영역이 §3-2에 명시됨
- 형에게 "여기까지 결론 + 추가 조사 시 바뀔 부분" 보고

미충족 → 추가 조사 영역 명시 + 형의 판단 요청.

**종료 필수 제출물:** §3-2 관찰 한계 + §3-5 축별 신뢰도

---

## 3-5. 축별 신뢰도 등급

각 축의 결론에 신뢰도를 태깅한다 — 모든 축의 결론이 같은 무게가 아니다.

> [!warning] 모드 분기 (먼저 확인)
> - **LIGHT 모드:** 아래 3요소 분해 절차 **전체 면제**. 축당 "높음/낮음" 1단어만 기록. 확신도는 **3요소 판정과 무관하게 50으로 cap**. 매핑 테이블 우회. DEEP 3요소 경로를 LIGHT에서 실행하면 오사용(T1 E5 FAIL).
> - **DEEP 모드:** 아래 3요소 분해 + 매핑 테이블 + 개수 세기 검증 풀 수행.

| 요소 | 높음 | 낮음 |
|------|------|------|
| 소스 다양성 | 3종+ 유형 | 1종 |
| 포화 달성 | 달성 | 미달 |
| 반대증거 | 탐색 후 반증 실패 (가설 강화) | 미탐색 or 반증 성공 |

**확신도 태깅 규칙 (필수):**
- 반드시 90/70/50/30 중 하나만 사용. 중간값(65, 75, 80, 85 등) 절대 금지. 100점 척도 금지.
- 3요소 각각을 "높음" 또는 "낮음" 이진 판정. "중간" "부분" 사용 금지. 애매하면 낮음으로 판정.
  - 소스다양성: 3종+유형 = 높음, 그 외 = 낮음
  - 포화: 달성 = 높음, 미달 = 낮음
  - 반대증거: 탐색 후 반증 실패(가설 강화) = 높음, 미탐색 또는 반증 성공 = 낮음
- "높음" 개수를 세어 확신도 결정:

| 높음 개수 | 확신도 |
|-----------|--------|
| 3 | 90 |
| 2 | 70 |
| 1 | 50 |
| 0 | 30 |

- 형식: "확신도: [90/70/50/30] — 소스다양성 [높음/낮음], 포화 [높음/낮음], 반대증거 [높음/낮음]"
- **확신도 태깅 위치:** §3-5에서만 태깅한다. §1·§2 본문에서 "(확신도: 80)" 또는 "확신도 추정: 60" 같은 자유 점수를 붙이지 마라. "확신도"라는 단어 자체를 §3-5 태깅 외부에서 수치와 함께 사용하지 않는다. 본문에서 신뢰도를 표현할 때는 "포화 달성/미달", "소스 3종 일치", "신뢰도 높음/낮음" 등 서술형으로만 기술한다.
- **개수 세기 검증:** 높음 판정 후 반드시 "소스다양성=[높음/낮음], 포화=[높음/낮음], 반대증거=[높음/낮음] → 높음 N개"를 명시적으로 나열한 뒤 매핑 테이블을 적용한다. 나열 없이 바로 확신도를 쓰지 마라.
