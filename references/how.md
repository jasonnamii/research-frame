조사를 실행하고, 편향을 방지하고, 교차하여 통찰을 도출한다.

## 2-1. 축별 독립 조사

축 간 오염 방지를 위해 독립성 유지 — A축 결과가 B축 방향에 영향 주면 독립성이 깨진다.

**병렬 실행 (MUST):** 독립 축의 WebSearch를 **병렬 tool call로 동시 발행**한다. 축 A·B·C가 독립이면 3개 WebSearch를 한 메시지에서 동시 호출. "병렬 또는 독립 순차"가 아니라 **병렬이 기본, 순차는 의존성 있을 때만**. 이유: 3축 직렬 → 3× wall-clock. 병렬 시 조사 시간 ~40%↓. 축 간 독립성도 자동 보장.

**절차:** 축당 독립 리서치(병렬) → 수렴검증4축(아래) → 반대증거 탐색(§2-2) → `_research/`에 저장

**수렴검증4축 (CONVERGENCE4):**
| 축 | 의미 | 예시 |
|---|---|---|
| 소스유형 | 1차(정부·IR·학술) / 2차(업계리포트) / 3차(일반미디어) 중 다른 유형 | 정부통계 + 업계리포트 |
| 출처거리 | 이해관계·소속이 다른 출처 | 기업IR + 독립애널리스트 |
| 접근법 | 정량/정성, 연역/귀납 등 다른 방법론 | 설문데이터 + 사례분석 |
| 시간 | 다른 시점의 데이터 | 2023년 + 2025년 |

- 축별 독립소스 1개↑ 확보 필수
- FULL 산출물 → 4축 (≤2축 → 확신도 1tier 강등)
- 단일팩트확인(단일주제·단일소스도메인) → 2축 충분
- LIGHT·정보전달 → 2축 충분
- 3차 소스 단독 → ✗ (1차/2차 교차 필수)

**가정 실시간 점검 (Key Assumptions Check, 축당 필수):**
각 축 조사 시 "이 축이 가정하는 것은?"을 명시하고, 새 증거 발견마다 "이 가정이 여전히 유효한가?" 실시간 점검한다. HOW 게이트의 전제 점검이 사후라면, 이것은 조사 중 조기 감지 메커니즘이다.
- 축 착수 시: 해당 축의 핵심 가정 1-2개 선언
- 새 증거가 가정을 훼손하면: 즉시 메모 + 축 완료 후 §2-4에서 판정
- LIGHT: 면제

**중간보고 (병렬 환경 트리거):**
- 병렬 배치 실행 시: 첫 번째 배치 완료 시점에 보고 (예: 4축을 2+2로 병렬 시, 첫 2축 결과 도착 시)
- 전 축 동시 완료(3축 이하 단일 배치): 중간보고 생략 → 교차분석으로 직행
- 보고 내용: 축 유효성·예상 못한 축·방향 전환 판단
- 중간보고 후 **축 추가/삭제/변경 가능** — 형 승인 시 §2-1 재진입 (신규 축은 기존 축과 병렬 또는 순차)

**포화 판정:** 새 소스가 결론을 변경하지 않는 것이 2회 연속(소스 유형이 달라야 함)이면 종료. 반대증거는 포화 카운트 제외 — 항상 추가 탐색.

**선행 교차:** 포화 축 2개↑ → 완료 축 간 L2 교차 선행 착수 가능("잠정" 태깅, §2-3에서 재수행).

**소스 fallback:** 상위tier 접근불가 → 차선tier 대체 + 사유 명시 + 확신도 1↓(@ref UP §E).

**리서치-기획 분리:** `_research/`와 기획안 분리. 도구 한계를 이유로 거부 금지, 가용 지식으로 착수 + 확신도로 한계 반영.

## 2-2. 편향 방지 체계

확증편향만 막으면 다른 편향이 결과를 오염시킨다.

**반대증거 탐색(필수):** 핵심 가설마다 "이 가설이 틀리다면 어떤 증거가 있을까?" 최소 1회 탐색.
- 반대증거 0건 → 의심 플래그 (확신도 하향 고려)
- 반대증거 발견 → 가설 수정 또는 조건 분기

**대안 시나리오 구성 (Devil's Advocacy, DEEP 필수):**
반대증거를 "찾기"만 하면 수동적이다. 각 반대증거에 대해 "만약 이것이 사실이라면?" 시나리오를 능동적으로 구성한다.
- 절차: 반대증거 발견 → "이 증거가 참이면 현재 가설이 어떻게 바뀌는가?" 1줄 시나리오 작성
- 시나리오가 설득력 있으면: 경쟁 가설(§1-2)에 추가 → 증거-가설 매트릭스(§2-3)에서 구조적 비교
- LIGHT: 면제

**편향 체크리스트(축별 완료 시, 위험 2개 선택):** 5종 전부 돌리면 오버헤드가 크므로, 해당 축에서 가장 빠지기 쉬운 2개를 골라 점검.

| 편향 | 점검 질문 |
|------|-----------|
| 확증 | 가설 지지 증거만 모았는가? |
| 생존 | 성공만 보고 실패·소멸 사례를 빠뜨렸는가? |
| 선택 | 접근 가능 소스에 편중되어 모집단 미대표? |
| 가용성 | 최근·인상적 사례에 과도한 비중? |
| 후지식 | 이미 알고 있는 결과에 끌려 자의적 해석? |

## 2-3. 교차분석

축별 결론만 나열하면 교차분석이 아니다 — 축 간 모순과 강화를 명시적으로 도출해야 통찰이 나온다.

**절차:**
1. 사전 점검: 전제를 흔든 증거 있으면 형에게 보고
2. **증거-가설 매트릭스 (ACH, DEEP 필수):** 경쟁 가설(§1-2)을 열, 핵심 증거를 행으로 배치. 각 셀에 일관성(C: Consistent / I: Inconsistent / N: Neutral) 판정. **진단성 판정:** 증거가 가설 간을 구별하는 능력. 모든 가설에 C인 증거 = 진단성 낮음(무시). C/I가 갈리는 증거 = 진단성 높음(핵심). 가장 적은 I를 가진 가설이 최적 후보. **행 상한: 6행** — 진단성 높은 증거 상위 6개만 기재. 7행 이상은 저진단성 증거가 섞여 매트릭스 가독성이 떨어지고 판정 노이즈가 증가한다. 7개 이상 후보가 있으면 진단성 점수로 상위 6개 선별.
3. 연역수렴: L1(축별 독립) → L2(축간 교차) → 모순·강화·빈자리 **각각** 명시. 0건 항목 있으면 재탐색. L1만으로 결론 금지
4. 정량↔정성 엇갈리는 지점 명시 — 핵심 통찰 후보
5. **결론 견고성 검증 (민감도 분석):** 최약점 3개 선택 → 각 반전 시나리오에서 "의사결정이 바뀌는가?" 추적. 바뀌면 = 핵심 불확실성으로 §3-5 신뢰도에 반영. 안 바뀌면 = 결론 견고.
6. 핵심 통찰 2-3개 도출

**확산 5유형 (프레임 밖 점검, DEEP 필수):**

| # | 유형 | 점검 질문 | 해당 시 조치 |
|---|------|-----------|-------------|
| 1 | 인접산업 | 유사 문제를 다른 산업에서 풀었는가? | 사례 축약 + 시사점 기록 |
| 2 | 반대사례 | 우리 가설과 반대 결론을 낸 사례가 있는가? | §2-2 반대증거에 편입 |
| 3 | 실패패턴 | 같은 방향으로 시도했다 실패한 사례가 있는가? | 실패 원인 + 우리와의 차이 기록 |
| 4 | 규제변화 | 규제·정책 변화가 결론을 뒤집을 수 있는가? | 시나리오 분기 + 타임라인 기록 |
| 5 | 기술전환 | 기술 변화가 전제를 무효화할 수 있는가? | 전제 훼손 경로 + 시점 추정 |

**유형당 사례 상한: 3개** — DEEP에서도 해당 유형에 사례가 많으면 영향도·확실성 상위 3개만 기재. 나열이 길어지면 핵심 판정이 흐려지고 교차분석 부담이 커진다.
- LIGHT: 유형당 해당/비해당 1줄 판정만 (조치·사례 전부 생략)

## 2-4. 가설 갱신 프로토콜

리서치 중 가설이 반증되는 것은 자연스러운 현상 — 행동 경로가 미정의되면 혼란이 온다.

| 상황 | 행동 |
|------|------|
| 핵심 가설 반증 + 전제 훼손 | §1 WHY 1-2 회귀 → 가설 재설계 → 프레임 재점검. **WHY 회귀 하드캡: max 2회.** 3회째 반증 시 회귀 대신 "가설 미확정" 태그 + 현재 증거 기반으로 §3 진행 |
| 부분 수정 (전제 유지) | §2 내 갱신 + 사유 기록 + 계속 |
| 새 가설 후보 발견 | `_research/가설메모.md` 기록 + 현 축 완료 후 검토 |

**순환 방지:** 기획 통찰 발생 → `_research/기획메모_[축명].md`에 기록, 상위 스킬(policy-planning 등) 역호출 금지. **WHY 회귀 루프 캡(max 2회)이 무한 순환의 1차 방어선.** 2회 회귀 후에도 가설이 불안정하면, 현재 증거 기반 진행 + 확신도 1tier 강등이 현실적.

## 2-4b. 프레임 리프레이밍 (DEEP, 조건부 발동)

가설 갱신(§2-4)은 기존 프레임 안에서 가설을 수정한다. 리프레이밍은 **프레임 자체**(축의 선택, 렌즈의 적절성)를 의심하고 대안 프레임을 탐색한다.

**발동 조건 (하나라도 충족 시):**
- 전제 훼손 + 축 간 모순 3개↑
- WHY 회귀 2회 소진 후에도 가설이 불안정
- 형이 "프레임이 이상하다" 명시

**절차:**
1. 현재 프레임의 근본 가정 1줄 정리: "우리는 [X]라고 전제하고 [Y] 축으로 보고 있다"
2. 대안 프레임 1-2개 탐색: "만약 [X]가 아니라 [Z]라면, 어떤 축으로 봐야 하는가?"
3. 대안 프레임 각각에 대해 §1-3 MECE 검증 약식 수행 (축 겹침/누락만)
4. 현재 프레임 vs 대안 프레임 → 형에게 선택 요청
5. 프레임 변경 시: 기존 조사 결과 중 재활용 가능분 식별 + 신규 축만 §2-1 재진입

**하드캡:** 리프레이밍 최대 1회. 2회째 발동 시 → "프레임 불안정" 태그 + 현재 최적 프레임으로 진행
**LIGHT:** 면제
