# §2 HOW — 어떻게 조사하는가

조사를 실행하고, 편향을 방지하고, 교차하여 통찰을 도출한다.

**v2 설계 원칙: 단일 지점 판정 + 포인터 재참조.** 반대증거·ACH·전제·신뢰도는 각각 단일 섹션에서만 판정. 중복 재판정 = FAIL.

> [!info] LIGHT 모드
> LIGHT는 경량화 블록만 따른다. 상세 절차는 DEEP 전용.
> - 2-1: 축당 핵심 소스 1~2개 scan. 수렴검증4축 면제. 전제 실시간 점검 면제. 산출: `_research/LIGHT_[축명].md`
> - 2-2: 반대증거 1회 시도만. 대안 시나리오·편향 체크리스트 면제
> - 2-3: L1만 (축별 방향성). L2 생략. ACH 면제. 확산 5유형은 scan 1줄
> - 2-4: 방향 전환 필요시 메모만
> - HOW 게이트: 면제

## 2-1. 축별 독립 조사

축 간 오염 방지를 위해 독립성 유지.

**병렬 실행 (MUST):** 독립 축 WebSearch를 **병렬 tool call로 동시 발행**. 직렬 = 3× wall-clock, 병렬 = ~40%↓.

**절차:** 축당 독립 리서치(병렬) → **홈즈**(장르별 관찰·침묵·비대칭 포착) + **엄브렐러**(5장르 MECE 배치) → 수렴검증4축 → 반대증거 재참조(§2-2) → `_research/`에 저장

**수렴검증4축 (CONVERGENCE4):**

| 축 | 의미 | 예시 |
|---|---|---|
| **소스유형** | 5장르 중 서로 다른 장르 | 공식 + 전문 + 커뮤니티 |
| 출처거리 | 이해관계·소속 다른 출처 | 기업IR + 독립애널리스트 |
| 접근법 | 정량/정성, 연역/귀납 등 다른 방법론 | 설문 + 사례분석 |
| 시간 | 다른 시점 | 2023년 + 2025년 |

**5장르 매트릭스 (소스유형 축, v2.5):**

| 장르 | 진실의 결 | 적합 주제 |
|---|---|---|
| **공식** (정부·IR·학술·법령) | 권위·지연·포장 | 규제·재무·정책 |
| **전문** (업계리포트·애널리스트) | 구조·해석 | 산업동향·M&A |
| **커뮤니티** (레딧·디시·블라인드) | 날것·패턴·분노 | UX불만·실사용·내부정황 |
| **소셜** (X·스레드) | 실시간·감정·소문 | 바이럴·이벤트·초기신호 |
| **역설** (퇴사자·경쟁사·피해자·반대진영) | 부정적 진실 | 리스크·구조적 결함 |

- 축별 독립소스 1개↑ 확보 필수
- FULL 산출물 → 수렴4축 (≤2축 → 확신도 1tier 강등)
- 단일팩트확인·LIGHT → 2축 충분
- **게임이론:** 상대가 숨기려는 정보는 반대편(역설 장르)에 — **프리모르템**(실패 역추적)으로 역설 장르 강제 포함 (R8)

**→ v2 직결:** 수렴4축 충족 = §3-5 "소스다양성" 판정에 **직결**. 재계산 금지. 매핑: 3축+ 충족 = 높음, 2축 이하 = 낮음.

**전제 실시간 점검 (단일 집행처):**
§1-3 선언 전제 P1/P2/P3을 조사 중 실시간 점검.
- 축 착수 시: 의존 전제 1-2개 식별
- 새 증거가 전제 훼손 시: 로그 기록 + 축 완료 후 §2-4 판정
- 조사 완료 시: 전제별 "유지/훼손" 태깅
- LIGHT: 면제

**중간보고:** 병렬 배치 실행 시 첫 배치 완료 시점. 전 축 동시 완료(3축 이하 단일 배치) = 생략. 축 추가/삭제/변경 가능(형 승인 시 §2-1 재진입).

**포화 판정:** 새 소스가 결론을 변경하지 않는 것이 2회 연속(장르가 달라야 함)이면 종료. 반대증거는 제외(항상 추가 탐색).

**소스 fallback:** 상위tier 접근불가 → 차선tier + 사유 명시 + 확신도 1↓. **맥가이버**(제약→가용자원 재해석·비표준용도)로 우회 경로 설계.

**리서치-기획 분리:** `_research/`와 기획안 분리. 도구 한계 이유 거부 금지.

**깊이 재판정 트리거 (B):** 축 조사 중 아래 2조건 중 하나면 §3-3 깊이 1단계 상향 or 축당 소스 2개+ 추가.
- ①핵심 수치·고유명사가 단일 소스 의존 (수렴4축 미충족)
- ②반대증거 1건↑ 발견 (§2-2 출력 참조)
- 축당 1회 상한. LIGHT 면제.

**FIG 수집 (C, 팩트 무결성 1단계):** 조사 중 판정 좌우 수치·고유명사·날짜·인용문을 팩트 장부에 누적. 반전위험7(단위·긍부정·시점·주어·비교방향·조건범위·고유명사) 태그 병기. 저장: `_research/[축명].md` § 팩트 장부. LIGHT: 근거 3개 한정.

## 2-2. 반대증거 허브 (단일 지점)

**단일 지점 원칙:** 반대증거 탐색·대안 시나리오·편향 점검은 이 섹션에서만 1회 수행. §2-3 확산 5유형의 "반대사례"와 민감도 분석은 이 허브 **재참조**만.

**반대증거 탐색:** **프리모르템**(실패 확정 전제·역추적·치명원인 Top3) 발동. 핵심 가설마다 "이 가설이 틀리다면?" 최소 1회.
- 반대증거 0건 → 의심 플래그 (§4 S3 연쇄)
- 발견 → 각각 대안 시나리오 구성

**대안 시나리오 (Devil's Advocacy):** 반대증거마다 "만약 사실이라면?" 1줄 구성.
- 설득력 있음 → §1-2 경쟁 가설 추가 → §2-3 ACH 편입
- 약함 → 각주 보관
- LIGHT: 면제

**편향 체크리스트 (축별 완료 시, 위험 2개 선택):**

| 편향 | 점검 질문 |
|------|-----------|
| 확증 | 가설 지지 증거만? |
| 생존 | 성공만·실패/소멸 누락? |
| 선택 | 접근 가능 소스 편중? |
| 가용성 | 최근·인상적 과도? |
| 후지식 | 결과에 끌린 해석? |

**허브 출력 (§2-3·§3-5 재참조용):** 반대증거 N건 · 대안 시나리오 M개 · 편향 점검 통과/미통과

## 2-3. 교차분석

축별 결론 나열 ≠ 교차분석 — 모순과 강화를 명시적으로.

**절차:**
1. §2-1 전제 실시간 점검 로그에서 훼손 징후 있으면 형에게 보고
2. **연역수렴 L2** 발동 → 축간 교차(모순·강화·빈자리) + ACH 매트릭스
3. **ACH (DEEP 필수):** 경쟁 가설(§1-2)을 열, 핵심 증거를 행(상한 6행, 진단성 상위). 각 셀 C(Consistent)/I(Inconsistent)/N(Neutral) 판정. 진단성 = 가설 간 구별력. 모든 가설에 C인 증거 = 진단성 낮음. C/I 갈리는 증거 = 핵심. 최소 I 가설 = 최적 후보.
4. 정량↔정성 엇갈리는 지점 명시 — 핵심 통찰 후보
5. **민감도 분석:** §2-2 허브 최약점 3개 선택 → 반전 시 "의사결정 바뀌는가?" 추적. **재탐색 금지** — §2-2 출력 재활용만. 바뀌면 = 핵심 불확실성 §3-5 반영.
6. 핵심 통찰 2-3개 도출
7. **FIG 결론 (C, 2단계):** 팩트 장부 3축 검증. **미통과 팩트는 결론에서 제외.**

**FIG 3축 — 오컴(가정최소)·외과적(반전위험 점검) 발동:**

| # | 축 | 점검 | 미통과 시 |
|---|-----|-----|-----------|
| 1 | 수치 정합 | 동일 수치가 2소스 이상 일치(단위·시점·주어)? (수렴4축 직결) | 재탐색 or 태그 + 신뢰도 1tier↓ |
| 2 | 단어 반전 | 반전위험7 중 해당 태그 재확인 — 원문 반전 여부 | 진술 수정 + 원문 재인용 |
| 3 | 자기 인용 | 결론이 LLM 생성 진술 순환 참조? | 외부 1차 소스 대체 or 삭제 |

**FIG 통과 전제:** §3-5 신뢰도 = FIG 통과 팩트만 입력. LIGHT: FIG 1축(수치 정합)만.

**확산 5유형 — 아날로지(타도메인 매핑) + 줌(스케일 전환) 발동:**

| # | 유형 | 점검 질문 | 조치 |
|---|------|-----------|------|
| 1 | 인접산업 | 유사 문제 다른 산업? | 사례 축약 + 시사점 |
| 2 | 반대사례 | 우리와 반대 결론? | **§2-2 허브 포인터** — 재탐색 금지 |
| 3 | 실패패턴 | 같은 방향 실패 사례? | 원인 + 우리와 차이 기록 |
| 4 | 규제변화 | 규제 변화가 뒤집나? | 시나리오 분기 + 타임라인 |
| 5 | 기술전환 | 기술 변화가 전제 무효화? | 전제 훼손 경로 + 시점 |

유형당 사례 상한: 3개. LIGHT: 유형당 해당/비해당 1줄만.

## 2-4. 회귀 절차 (수정4 L0~L4 재활용)

가설·프레임이 흔들리는 건 자연스러움 — 행동 경로 미정의 시 혼란. **수정4** 트리거의 L0~L4 레벨 판정을 재활용한다.

**진입 조건:** 핵심 가설 반증 / 전제 훼손 / 축간 모순 3개↑ / 형 "프레임 이상하다" 명시

**수준 판정 (진입 시 1줄):**

| 기준 | 수준 | 회귀 대상 |
|-----------|------|-----------|
| 프레임 유효, 가설만 흔들림 | **가설 수준** | §1-2 |
| 전제 훼손 + 축간 모순 3개↑ / 형의 프레임 비판 | **프레임 수준** | §1-3 |
| 애매 | 가설 수준 먼저 → 미해결 시 프레임 수준 승격 | — |

**가설 수준 회귀:**
1. §1-2 회귀 → 가설 재설계 (§2-2·§2-3 반영)
2. ACH 매트릭스 갱신
3. 영향 축만 §2-1 부분 재진입

**프레임 수준 회귀 (틀밖 발동):**
1. 근본 가정 1줄: "우리는 [X]라고 전제하고 [Y] 축으로 본다"
2. **틀밖**(프레임 박제·거부·3대 킬샷) → 대안 프레임 1-2개 탐색
3. 대안 프레임 각각 §1-3 MECE 약식 수행
4. 현재 vs 대안 → 형 선택
5. 프레임 변경 시: 재활용 가능분 식별 + 신규 축만 §2-1 재진입

**통합 하드캡:** 총 3회 (가설 2회 + 프레임 1회). 소진 시 "회귀 한계 도달" 태그 + 현재 상태로 §3 진행 + 확신도 1tier 강등.

**순환 방지:** 기획 통찰 → `_research/기획메모_[축명].md`, 상위 스킬 역호출 금지.

**부수 처리:**

| 상황 | 행동 |
|------|------|
| 부분 수정 (전제 유지) | §2 내 갱신 + 사유 기록 |
| 새 가설 후보 | `_research/가설메모.md` + 현 축 완료 후 검토 |

**LIGHT:** 전 절차 면제.
