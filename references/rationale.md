# Design Rationale — 왜 이렇게 설계했는가

본 스킬의 게임이론·메커니즘 디자인 배경. 본문 규칙을 읽을 때 "왜 이 규칙?"이 궁금하면 여기만 읽는다. 본문은 규칙만, 이 파일은 이유만.

---

## R1. 맹점 자기고발 = dominant strategy (§4 BLIND)

**본문 규칙:** 발견 = 보너스(회귀 +1), 숨김 = 페널티(신뢰도 2tier↓), 미발견 = 무페널티.

**왜 비대칭인가:**
- 탐지 실패에 페널티 주면 → Claude가 "아예 찾지 않는 쪽"으로 학습 (가짜 음성 우선)
- 발견 보너스 + 숨김 페널티만 → 성실 탐색 + 정직 보고가 우월 전략
- **Nash Equilibrium:**
  - 전략 A (적극 탐색·정직 보고): 보너스↑, 페널티 0
  - 전략 B (소극 탐색·정직 보고): 보너스↓, 페널티 0
  - 전략 C (탐색·숨김): 보너스 0, 페널티 높음 → 지배적으로 열등
- **균형:** A. 넛지 트리거(선택자유 보존·디폴트 설계·의도방향 유도) 적용.

---

## R2. EXPAND 셸링 포인트 = 주가설 복귀 (§5)

**본문 규칙:** 애매할 때 기본 행동 = 주가설 복귀. 확장은 명시적·예산 내·성공조건 명확할 때만.

**왜 "수렴"을 기본값으로 고정하는가:**
- 반복게임에서 양쪽(확장 vs 수렴)이 배신하면 리서치 붕괴
- 기본값 = 수렴으로 고정하면 확장은 "고의적 고비용 선택"이 됨 → 남용 차단

**셸링포인트 적용 지점:**
| 상황 | 기본 행동 |
|---|---|
| 애매함 | 주가설 복귀 (확장 중단) |
| 증거 혼재 | 주가설 우선 (프레임밖 = 각주) |
| 시간 부족 | 주가설 완료 우선 (프레임밖 skip) |
| Claude·형 의견 대립 | 형 판정 (맹점 로그 기록) |

→ 트리아지 트리거의 결과물.

## R3. 복귀 서약 = Commitment Device (§5-7)

**본문 규칙:** 토큰 사용 전 4항(토큰·사유·가설·복귀조건) 선언 의무.

**왜 사전 선언인가:**
- 사용 전 종료 조건 박아두면 → 중간에 마음 바꿀 수 없음
- "조금만 더" "감이 왔어" = 주관적 연장 시도 차단
- 3턴·수렴4축 2축 = 객관 메트릭만 허용

→ 타임스톤 트리거 적용.

---

## R4. 연쇄 토큰 = 별도 예산 (§5-8)

**본문 규칙:** §4 감지 → §5 자동 발급 토큰은 정규 예산에서 차감 ✗.

**왜 별도 예산인가:**
- 맹점 감지 = 성실 탐색의 결과
- 정규 예산 차감 시 → 맹점 발견 회피 유인 발생
- 별도 예산 = 맹점 적극 탐지 보상 (R1과 정합)

→ 넛지 + 프리모르템 트리거 조합.

---

## R5. 수렴4축 직결 = 중복 회계 제거 (§3-5)

**본문 규칙:** §3-5 "소스다양성"은 §2-1 수렴4축 충족 수를 직결 매핑. 재계산 ✗.

**왜 직결인가:**
- 같은 증거 재계산 → 시간 팽창 + 판정 불일치
- 단일 지점 판정 + 포인터 재참조 = v2 핵심 원칙
- 오컴 트리거 적용.

**직결 매핑:** 수렴4축 3축+ = 소스다양성 높음 / 2축 이하 = 낮음

---

## R6. LIGHT 확신도 cap 50 (INVARIANT 4)

**본문 규칙:** LIGHT 모드는 3요소 판정과 무관하게 50 절삭.

**왜 하드캡인가:**
- LIGHT = "폭 유지·깊이 최소". 우연히 3/3이라도 깊이 부족
- cap 없으면 LIGHT가 DEEP 대체재로 오남용 → DEEP 존재 이유 소멸

→ 외과적 트리거 적용.

---

## R7. 트리거 연동 = 의미론 외장화 (INVARIANT 9, v2.5)

**본문 규칙:** research-frame 본문은 트리거 정식명을 호출 1줄로 사용. 의역·변형 ✗.

**왜 트리거에 의미론을 위임하는가:**
- 사고도구(홈즈·엄브렐러·연역수렴 등)는 trigger-dictionary가 정의·유지
- research-frame이 같은 사고를 재설명하면 → 이중 유지보수, 정의 drift
- 본문에 정식명 박으면 → trigger-dictionary §0-0 HARD-FIRE가 자동 스캔·호출

→ 스켈레톤 트리거(비본질 제거·뼈대만 잔존) 적용 결과.

**결합도 비용:** trigger-dictionary 개정 시 research-frame 회귀 테스트 필수. 이 비용이 이중 유지보수 비용보다 작다는 판단.

---

## R8. 5장르 소스 매트릭스 = 홈즈+엄브렐러 (v2.5)

**본문 규칙:** 수렴4축 "소스유형" 축의 내용물을 1·2·3차 계층 → 5장르(공식·전문·커뮤니티·소셜·역설)로 재설계.

**왜 5장르인가:**
- 1·2·3차 계층은 "권위 순서"만 표현 — 진실의 **결**을 못 구분
- 공식(권위·포장) / 전문(구조·해석) / 커뮤니티(날것·패턴) / 소셜(실시간·감정) / 역설(부정적 진실) = 각기 다른 진실의 결
- 사람이 리서치하듯, **어디를 볼지**를 먼저 맞히게 함 (홈즈)
- 5장르로 우산 씌움 (엄브렐러)

**게임이론:** 상대가 숨기려는 정보는 반대편(역설채널)에 있음 → "역설" 장르 강제 포함이 이 비대칭을 이용.
