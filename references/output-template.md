# 산출물 내부 구조 템플릿

축별 파일(`_research/[축명].md` 또는 `_research/LIGHT_[축명].md`) 안에 들어갈 섹션 순서와 필수 항목. 피라미드 원칙(결론 선행) + 인텔리전스 브리핑(증거 계층화) + PRISMA(투명성)를 반영.

---

## DEEP 템플릿 — `_research/[축명].md`

```markdown
# [축명]: [핵심 주제]

## 핵심 판정
**[결론 1문장]** — 의사결정자가 가장 먼저 읽어야 할 내용.
의사결정 연결점(§1-1)과의 관계: "이 판정이 [의사결정]에 [어떻게] 영향한다"

## 근거 체계

### 주 근거 (Primary Evidence)
- 출처1 [URL]: [사실/수치] → [판정에 미치는 함의]
- 출처2 [URL]: [사실/수치] → [판정에 미치는 함의]

### 보조 근거 (Corroborating Evidence)
- [추가 확인 요소들]

### 수렴검증4축 충족 현황
| 축 | 충족 여부 | 소스 |
|---|---|---|
| 장르 (공식·전문·커뮤니티·소셜·역설 중 3종+) | ✓/✗ | [소스명 + 장르 라벨] |
| 출처거리 (1·2·3차) | ✓/✗ | [소스명] |
| 접근법 (정량·정성·비교·반대) | ✓/✗ | [소스명] |
| 시간 (과거·현재·미래 신호) | ✓/✗ | [소스명] |

## 반대증거 + 대안 시나리오
- 반대증거: [내용] (출처: [URL])
- 대안 시나리오: "만약 이것이 참이면 → [시나리오]"
- 판정: 기각 / 경쟁 가설에 편입 / 조건 분기

## 팩트 장부 + FIG 판정
| 팩트 | 출처 URL | 장르 | 반전위험 태그 | FIG 판정 |
|------|----------|------|---------------|----------|
| [수치/고유명사/날짜/인용문] | [URL] | 공식/전문/커뮤니티/소셜/역설 | 단위/시점/주어/비교방향 중 해당 | 통과/재확인/배제 |

**FIG 3축 요약:** 수치정합 [통과/미통과] · 단어반전 [통과/미통과] · 자기인용 [통과/미통과]
미통과 팩트는 §3-5 신뢰도 계산에서 배제됨.

## 맹점 로그 (§4 BLIND)
- S1 비대칭: [감지 YES/NO, 축별 증거수, 판정]
- S2 부재: [ACH 빈칸 수, 원인, 조치]
- S3 반증공백: [재탐색 결과, 판정]
- §5 EXPAND 연쇄: [T1/T2 자동 발급 여부]


## 5장르 발견 대비표 (S2 SOURCE_CONTRAST, v2.7)

| 장르 | 핵심 발견 1줄 | 1차 출처 (YYYY-MM) | 다른 장르와의 합의·모순 |
|------|---------------|---------------------|----------------------|
| 공식 | | | |
| 전문 | | | |
| 커뮤니티 | | | |
| 소셜 | | | |
| 역설 | | | |

합의 영역: [N장르 일치] · 모순 영역: [어느 장르가 어디서 갈리는가] · 빈자리: [미수집 장르]

## 평가·등급 근거ID 매핑 (S3 CLAIM_PROVENANCE, v2.7)

별점·강중약·우열·즉사 등 *모든 평가 등급*에 근거 ID(팩트 장부 행번호) 명시 의무.

| 평가 항목 | 등급 | 근거ID (팩트 장부 #) | 1줄 사유 |
|----------|------|---------------------|----------|
| [예: 양평DNA 활용도] | 강 | F3·F7 | 두물머리 + 황순원 = 단일 정체성 |
| [예: 마케팅 효율] | 높음 | F12 | 1브랜드 3채널 구조 |

근거ID 0개 = 평가 무효·FAIL.

## 본문 인용 의무 체크 (S1 EVIDENCE_BODY, v2.7)

| 체크 | 기준 | 결과 |
|------|------|------|
| 결론 한 줄당 1차 출처 1+ | §0 핵심 판정·각 §결론 | ○/✗ |
| 본문에 원자료(수치·인용문·고유명사) 포함 | §1·§2·§3 본문 | ○/✗ |
| "근거 체계" 섹션 외에 본문 ≥3회 출처 인용 | 본문 전체 | ○/✗ |

3개 모두 ○ = 통과. 1개+ ✗ = INV 10 위반·재작성 의무.

## 양적 포화 N+ 체크 (B1, v2.7)

| 깊이 | 1차인용 N | 충족 |
|------|-----------|------|
| 전수 ≥15 / 표본 ≥8 / 스캔 ≥3 | [실제 N] | ○/✗ |

미달 시 결론 어조 "추정" 강제 (B3 직결).


## 한계
- [접근 불가 소스 / 시간 부족 / 도구 한계]
- 각 한계가 판정에 미치는 영향: 치명적 / 경미
- 맥가이버 대체경로: [대체 소스·우회 기법]

## 가정 점검
- 이 축의 핵심 가정: [내용]
- 조사 후 판정: 유지 / 훼손
```

## 규칙

- **결론 선행 필수:** "핵심 판정" 섹션이 파일 최상단. 읽는 사람이 스크롤 없이 결론을 봐야 함
- **근거 계층화 필수:** 주 근거 → 보조 근거 순. 낱말 나열 금지 — 각 근거에 "판정과의 관계" 명시
- **장르 라벨 필수 (v2.5):** 모든 근거에 5장르(공식·전문·커뮤니티·소셜·역설) 라벨. 장르 누락 = FIG 미통과 간주
- **대안분석 필수 (DEEP):** "한계와 대안 시나리오" 섹션 없는 DEEP 산출물 = 미완성
- **맹점 로그 필수 (DEEP, INV 7):** 감지 여부와 무관하게 S1·S2·S3 판정 기록
- **URL 필수:** 모든 근거에 출처 URL. URL 없는 근거 = 미검증으로 표기
- **LIGHT는 경량 유지:** 3섹션(판정·근거·주의사항) 초과 금지

---

- **본문 인용 필수 (v2.7, INV 10):** 결론 한 줄당 1차 출처 1+ 인용. 표·등급만 있고 본문 인용 ✗ = FAIL
- **5장르 대비표 필수 (v2.7, INV 11):** 단일 장르 결론 = FAIL. 5장르 발견 대비표 미작성 = FAIL
- **근거ID 매핑 필수 (v2.7, INV 12):** 별점·강중약·우열 등 평가 옆 근거ID 누락 = FAIL
- **양적 포화 N+ 필수 (v2.7, B1):** 깊이별 N 미달 시 "추정" 어조 강제. 단정 = FAIL
- **출처 시점 필수 (v2.7, B5):** 모든 출처에 (YYYY-MM) 명시. 누락 = FIG 미통과
- **1차 출처 라벨 필수 (v2.7, B2):** 모든 팩트에 1차/2차/3차 라벨. 3차 단독 결론 = FAIL

---

## 경량화 규칙 (분량 상한)

분량 팽창은 판정 가독성을 훼손하고 의사결정 지연을 유발한다. 각 섹션에 상한을 두고, 초과분은 진단성·영향도 기준으로 선별하여 절사한다.

| 섹션 | 상한 | 초과 시 처리 |
|------|------|-------------|
| 주 근거 (Primary Evidence) | **2개** | 진단성·판정 기여도 상위 2개만 주 근거. 나머지는 보조 근거로 이동 |
| ACH 매트릭스 행(증거) | **6행** | 진단성 상위 6개 선별 (how.md §2-3 연동) |
| 장르별 팩트 (FIG 장부) | **장르당 3개** | 영향도·확실성 상위 3개 |
| 반대증거 + 대안 시나리오 | 핵심 가설당 **1-2개** | 시나리오 설득력 높은 것 우선, 나머지는 각주로 |
| LIGHT 근거 3개 | **고정 3개** | 초과 시 DEEP으로 모드 전환 고려 |

**원칙:** 상한은 하한이 아니다 — 진단성·영향도가 낮으면 상한 미달도 허용. 단, LIGHT 근거 3개는 고정.
