# TURBO 모드 — 고품질 병렬 가속화

**목적:** 리서치 시간 병목 해소. DEEP 품질 기준·INVARIANT·게이트 불변, §2-1·§2-2 축별 작업만 Agent 분산.

**트리거:** "터보 리서치 해줘" · "TURBO" · "리서치 터보" 명시. 미명시 = DEEP.

---

## 4원칙

1. **병렬화만, 스킵 금지** — INVARIANT(게이트·확신도 cap·반대증거 단일지점 등) 불변
2. **독립성 확인** — §2-1은 이미 "축 병렬 발행 MUST" 명시. Agent로 실체화
3. **통합 시퀀셜** — §2-3 교차분석·§2-4 회귀·§3 SCOPE는 메인이 수행
4. **Agent 브리프 완결성** — 축명·가설·수렴검증4축(장르 5분류 포함)·FIG 수집 지시·출처 위계·전제 목록 전체 전달

---

## 병렬 타겟

| 단계 | 병렬 단위 | 병렬도 |
|---|---|---|
| **§1-3 축 생성 (v2.5 옵션)** | 축 후보군 제안 Agent 1개 (MECE 맹점방어용, **틀밖·절대자** 자기반박 수행) | 1 |
| §2-1 축별 독립 조사 | 축마다 Agent 1개 (WebSearch·수렴검증4축·5장르·FIG 수집) | 축 수만큼 |
| §2-2 축별 반대증거 탐색 | 축마다 Agent 1개 (단일지점 원칙 유지) | 축 수만큼 |
| §4 BLIND 축별 S1·S2·S3 | 축마다 Agent 1개 (홈즈·절대자·오컴 발동) | 축 수만큼 |
| §5 EXPAND T1·T2·T3 | 토큰마다 Agent 1개 (틀밖·아날로지·줌) | 예산 한도 |

**§1-3 축 생성 Agent (v2.5 신설):** MECE 맹점(축 생성 편향) 방어용. 메인이 제안한 축 세트를 Agent가 **절대자**로 자기반박하여 누락 가능성 2개+ 도출. EXPAND 모드에서 필수, TURBO에서 옵션.

## 병렬 제외 (시퀀셜)

- §1-1·§1-2 WHY — 프레임 확정 전 §2 진입 ✗
- §2-3 교차분석·ACH·FIG 결론 — 축간 통합 필요
- §2-4 회귀 — 가설 흔들림 감지 → 재계획
- §3 SCOPE 전체 — 경계·한계·종료·신뢰도 판정
- INVARIANT·PREFLIGHT·RESET·STEALTH·WHY/HOW 게이트

---

## 품질 저하 방어

- 병렬 결과 교차검증 — 축간 용어·전제·결론 충돌 검출
- 축 간 오염 감지 (A 결과가 B에 유입) → 해당 축 Agent 재발행 1회 한
- FIG 수집 누락·확신도 정합성 실패 → 시퀀셜 재실행
- 불일치 미해소 시 TURBO 중단 → DEEP 시퀀셜 전환

## Agent 브리프 템플릿

축별 Agent 발행시 아래 요소 필수 전달:

1. **축명·범위** — 이 축이 다루는 질문·경계
2. **가설 3조건** — §1-2에서 확정된 가설·반증가능·구체성·검증경로 (**백본** 적용)
3. **전제 목록** — §1-3 전제 선언
4. **수렴검증4축** — UP §E 참조 + **장르 5분류**(공식·전문·커뮤니티·소셜·역설)
5. **FIG 수집 의무** — 3축 판정 데이터 수집 (**오컴·외과적** 적용)
6. **확신도 상한** — DEEP 판정 테이블 적용
7. **산출 경로** — `_research/[축명].md`
8. **상호참조 금지** — 다른 축 결과 참조 ✗
9. **트리거 정식명 발동 의무** — 응답 본문에 해당 층 트리거 노출 (홈즈·엄브렐러 등, §6 마스터 매핑)
