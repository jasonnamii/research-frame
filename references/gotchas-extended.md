# Gotchas — Research Frame 전체 함정 목록

허브(SKILL.md Top7) 외 전체 함정 + 섹션별 함정 통합. 각 references/ 파일은 이 파일로 포인터.

---

## 설계 함정

### 중복 회계 함정 (v2 핵심)
반대증거·ACH·전제점검·신뢰도를 여러 섹션에서 재판정 → 같은 증거 재소비로 시간 팽창.
- **단일 지점:** 반대증거=§2-2, ACH=§2-3, 전제=§2-1 실시간, 신뢰도=§3-5(수렴4축 직결)
- 다른 섹션은 **포인터 참조만** (오컴 트리거)

### 프레임 미확정 착수
WHY 게이트 없이 조사 시작 → 방향 전환 시 전량 재작업.
- [WHY 게이트] 3개 채움 후 §2 진입

### 축 간 오염
A축 결과가 B축 방향에 영향 → 독립성 깨짐.
- **독립 축은 WebSearch 병렬 발행 필수**
- 중간보고 전까지 축 간 정보 차단

### L1만으로 결론
축별 나열 ≠ 교차분석. **연역수렴 L2**에서 모순·강화·빈자리 명시 필수.

### 포화 오판
같은 출처 계열 2개 일치 ≠ 포화. 5장르 중 다른 장르여야 함.

## 프로세스 함정

### WHY를 거부기로 사용
"연결점 없어서 못합니다" = 절대자 위반. 추정 후 진행이 원칙.

### 무한 리서치
§3-1 경계 + §3-3 깊이 없이 전수 조사 시도.
- **회귀 통합 하드캡 총 3회** (가설 2회 + 프레임 1회)

### 관찰 한계 누락
깔끔 마무리 충동에 한계 생략 → 신뢰도 과대평가. §3-2 필수.

### 단일 가설에 매몰
§1-2 **경쟁 가설 3개+ 도출 필수** (ACH 원칙).

### 축 MECE 미검증
§1-3 MECE 체크리스트 필수 — 축 간 중첩·누락 확인. **틀밖 + 절대자**로 자기반박.

### 증거 진단성 무시
§2-3 진단성 판정 필수 — 모든 증거 동일 무게 취급 금지.

## 모드 함정

### LIGHT에서 DEEP 프로토콜 적용
LIGHT인데 포화 판정·중간보고·L2 교차 수행 → 오버헤드만 증가.
- references/ 각 파일의 LIGHT 블록만 따른다

### LIGHT 결과를 DEEP 수준으로 신뢰
LIGHT 확신도 상한 50. **cap 필수** — 3요소 판정 결과가 50 초과해도 50으로 절삭.

### LIGHT→DEEP 전환 시 파일 이중 존재
rename(mv) 사용, 복사 금지. LIGHT 결과는 "§초안" 섹션으로 보존(삭제 금지).

## 산출물 함정

### 산출물 템플릿 미준수
output-template.md 준수 필수.

### 통합본 작성
축별 분리 원칙 — 통합 금지. 기획안은 프로젝트 루트, 리서치는 `_research/[축명].md`.

### LIGHT/EXPAND prefix 누락
LIGHT 모드 = `LIGHT_` prefix, EXPAND 토큰 = `EXPAND_` prefix. 누락 시 혼동·오염.

## 팩트 함정 (v2.1)

### 팩트 오도
수치·고유명사가 단일 소스 의존인데 깊이 재판정(B) 트리거 미가동.
- §2-1 깊이 재판정 2조건 확인 필수

### FIG 미가동
팩트 장부 수집(§2-1)이나 3축 판정(§2-3) 누락 시 §3-5 신뢰도 과대평가.
- 장부 없음 = FIG 미통과 = 배제

### 자기 인용 순환
LLM이 조사 중 생성한 진술을 다시 근거로 삼는 순환 참조. FIG 3축(자기 인용) 필수 점검.
